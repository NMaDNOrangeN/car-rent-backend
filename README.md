# Система аренды автомобилей (Вариант 7)
## Описание

Backend-приложение для системы аренды автомобилей.

Проект реализует REST API, позволяющее:
- клиентам — просматривать каталог автомобилей, оформлять аренду и оставлять отзывы;
- менеджерам — управлять автопарком (бренды, модели, машины) и арендами;
- администраторам — управлять пользователями и их ролями.

## Функционал

- Регистрация не предусмотрена — вход осуществляется по предзаполненным учётным записям.
- JWT-аутентификация и ролевая авторизация (`admin` / `manager` / `client`).
- CRUD-операции для всех сущностей: пользователи, бренды и модели авто, автомобили, аренды, отзывы.
- Пагинация и фильтрация при получении списка автомобилей и аренд.
- Расчёт стоимости аренды с автоматической скидкой за длительный срок.
- Валидация входных данных на уровне Pydantic-моделей и бизнес-логики.
- Логирование значимых действий в файл `logs.log`.
- Стартовое заполнение БД (`seed.py`) тестовыми пользователями, машинами и арендами.

### Доп. задание — расчёт стоимости со скидкой за длительную аренду

При создании или изменении сроков аренды стоимость рассчитывается автоматически (`calculate_rental_price`) с применением скидки в зависимости от количества дней аренды:

| Срок аренды | Скидка |
|---|---|
| до 5 дней | без скидки |
| от 5 дней | 5% |
| от 10 дней | 10% |
| от 20 дней | 20% |

## Запуск

1. Создание venv (один раз при установке проекта):\
`python -m venv .venv`

2. Активация venv:\
`.venv\Scripts\activate`

3. Установка зависимостей:\
`pip install -r requirements.txt`

4. Создание файла `.env` в корне проекта по примеру `.env.example` и заполнение переменными

5. Создание БД и загрузка посева\
`python seed.py`

6. Запуск проекта:
    1. В режиме dev\
    `python -m fastapi dev main.py`
    2. В режиме prod\
    `python -m fastapi run main.py`

## API

- **Методология:** REST.
- **Формат передачи данных:** JSON.
- **Аутентификация:** JWT Bearer-токен.
- **Документация по всем эндпоинтам:** FastAPI (Swagger UI):\
`http://127.0.0.1:8000/docs`.
- **Пагинация и фильтрация:** query-параметры:
```
GET /api/cars?page=1&limit=10&type=sedan&minYear=2020&status=available
GET /api/rentals?page=1&limit=10&status=active
```

## Валидация

- **Email** — проверяется регулярным выражением (формат `user@example.com`).

- **Телефон** — проверяется регулярным выражением, допускается `+` и 10–15 цифр.
- **Пароль** — минимум 8 символов, только латинские буквы и цифры.
- **Год выпуска автомобиля** (`year`) — должен быть в диапазоне от 1900 до текущего года (проверяется как при создании, так и при обновлении автомобиля).
- **Оценка отзыва** (`rating`) — целое число от 1 до 5.
- **Комментарий к отзыву** — не более 200 символов.
- **Уникальность:** username, email, телефон пользователя; название бренда; название типа/статуса автомобиля и статуса аренды; пара (название модели, бренд) — модель с одинаковым названием не может повторяться в рамках одного бренда, но допускается у разных брендов; один отзыв на одну аренду.
- **Бизнес-валидация** (помимо Pydantic): проверка существования связанных сущностей при создании/обновлении (например, нельзя создать автомобиль с несуществующим `model_id`/`type_id`, нельзя создать аренду на несуществующего пользователя или автомобиль), проверка доступности автомобиля перед арендой, проверка корректности диапазона дат аренды (дата окончания не раньше даты начала), проверка завершённости аренды перед созданием отзыва.
- Все ошибки валидации возвращаются с соответствующим HTTP-статусом: `422 Unprocessable Entity` — для ошибок формата/типа данных (Pydantic), `400 Bad Request` — для нарушений бизнес-правил (дубликаты, несуществующие связанные сущности, недопустимые переходы состояний), `404 Not Found` — если сущность не найдена, `403 Forbidden` — при попытке доступа без необходимых прав или к чужим данным.

## Роли и права

| Роль | Права |
|---|---|
| **Клиент** (`client`) | Просмотр каталога автомобилей; создание аренды на доступный автомобиль; просмотр и управление только своими арендами и отзывами (отзыв можно оставить только после завершения аренды). |
| **Менеджер** (`manager`) | Полный CRUD автомобилей, брендов, моделей и аренд; просмотр всех отзывов и справочников (типы/статусы автомобилей, статусы аренды). Не имеет доступа к управлению пользователями. |
| **Администратор** (`admin`) | Полный CRUD пользователей и их ролей. Не имеет доступа к управлению автомобилями/арендами (это зона ответственности менеджера). |

Доступ к защищённым маршрутам определяется по роли (`type_id`) текущего пользователя на основе JWT-токена; маршруты, изменяющие данные или дающие доступ к чужой/конфиденциальной информации, защищены соответствующими зависимостями (`get_current_user`, `get_current_manager_user`, `get_current_admin_user`).
